WCA - Functional LCWRA, better task list tagging

If an LCWRA descriptor is chosen then the assessment should be curtailed. These versions direct the HCP to a page that tells them LCWRA has been met. This version contains a task list status of 'in progress' if any data has been entered in any descriptor pages. This carries through to the section level on the overview page. Descriptors are only tagged complete if both descriptor and justification are done. Sections are only tagged complete if every descriptor and justifcation is done - this may be less than ideal as they may not score against everything (if we do this, it should roll out to PIP too).

Versions

  • Version A (Straight to time review)
  • Version B (Option to go to time reviews)

PIP Version 0.2 (PROBE questions)

This prototype includes all of the questions from the PROBE v 3 document, and the scoring questions stated in the legislation. It works though the initial questions, which are then checked and confirmed before the assessor can do the scoring (done after the claimant has left).

Notes:

  • Research from Team Lime informed the choice of task lists for navigation, inline with GDS patterns. Left hand navigation was preferred by assessors however they also lost work because of the way it works, they were not realising the need to save answers. They could all use the task list without losing data, and no one had any problems with it. We think the task list is the viable choice for April, with the potential to further research and test left hand navigation if necessary for the future.
  • All inputs during the assessment are free text, with the exception of those legislated for scoring. This will be iterated in the next iteration to break them down and give built in structure where needed, and generate better quality data in return.
  • The guidance from the Probe document is shown underneath the free text fields.
  • The notes taken with the claimant present are displayed back to the assessor while they do the scoring questions for reference.
